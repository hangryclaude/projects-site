Like the Tab-autocomplete in Cursor/Copilot — but in *every* app. Start typing in any text field (Messages, Mail, Slack, Notes, a chat box, anywhere), and Ghostype floats a dim "ghost" of what you're about to say next at your caret. Press **Tab** to accept it. Predictions are generated by the shared AI controller (`~/Tools/lib/ai.cjs`) — free models first, paid fallback — and your *learned* writing (phrase model + style profile) stays on-device in `~/.ghostype/`.

```
You type:   Hey, just wanted to follow up on the ▌
Ghost:      Hey, just wanted to follow up on the ░status of my request░ ⇥
Press Tab → Hey, just wanted to follow up on the status of my request▌
```

## It learns *your* voice 🧠

Ghostype gets more like you the more you type. Everything stays **100% on your machine** in `~/.ghostype/` — nothing is ever uploaded.

- **Your own phrases (instant):** a tiny on-device n-gram model learns the things you actually write. If you always say "talk soon!" or "lmk when you're free," it completes them with **zero latency** in your exact wording.
- **Your style (AI-distilled):** every ~20 messages, the controller re-reads your recent writing and distills a short *style guide* — your tone, slang, emoji habits, punctuation, sign-offs. That profile is injected into every prediction so the model writes as you, not as a generic assistant.
- **Accepted = endorsed:** every suggestion you Tab-accept is logged as a strong signal and folded back into the model.

Real example after learning a casual texter's history:

| You start typing | Generic model | **Ghostype (your voice)** |
|---|---|---|
| sorry i cant make it tonight, ill | be thinking of you | **be there around 8ish lol** |
| hey do you want to | go out tonight? | **hang out this wknd?** |

**Menu bar controls:** see how many messages it's learned · toggle "Learn from my typing" · "Re-learn my style now" · "Clear learned data". Password fields are always skipped — never read, never logged.

## How it works

1. **Reads** the focused text field across any app via the macOS Accessibility API (`AXUIElementCreateSystemWide` → focused element → value + caret offset).
2. **Watches** your keystrokes with a `CGEventTap`; after you pause ~450 ms it grabs the text *before* the caret.
3. **Predicts** the continuation via the shared AI controller (`fast` tier by default — free GitHub Models, paid Anthropic fallback). Instant repeats of your own frequent phrases come from the on-device n-gram model with zero latency.
4. **Floats** the prediction as dim ghost text at the exact caret position (a borderless, click-through, non-activating panel).
5. **Accepts** on **Tab** — it swallows the Tab so you don't jump fields, then types the suggestion in at the caret. **Esc** dismisses.

## Install

**Prerequisite:** the shared AI controller at `~/Tools/lib/ai.cjs` with a working key in `~/.ai.env` (a free GitHub token is enough). Ghostype auto-starts the brain on launch (`node ~/Tools/lib/ai.cjs --serve 7611`); verify it with:
```bash
node ~/Tools/lib/ai.cjs --ping     # → free models OK
```

**Option A — download the app:** grab `Ghostype.dmg` from [Releases](../../releases), drag **Ghostype** to Applications, open it.

**Option B — build from source:**
```bash
git clone https://github.com/hangryclaude/ghostype.git
cd ghostype
./run.sh            # builds, signs, and launches Ghostype.app
```

A 👻 icon appears in your menu bar. First launch, grant **two** permissions:

**System Settings → Privacy & Security →**
- **Accessibility** → enable **Ghostype** (read the focused field + accept suggestions)
- **Input Monitoring** → enable **Ghostype** (watch your typing)

Then **quit Ghostype** (menu bar → Quit) and run `./run.sh` again. That's it.

## Use

- Type in any app. Pause briefly → ghost text appears at your caret.
- **Tab** = accept · **Esc** = dismiss · keep typing = it re-predicts.
- Menu bar icon: toggle predicting on/off, pick the model, open settings.

## Models (menu bar → Model)

| Tier | Routes to | Cost | Notes |
|-------|-------|---------|-------|
| `fast` | GitHub Models `gpt-4o-mini` | free | **default** — best for inline latency |
| `smart` | GitHub Models `gpt-4o` | free | sharper continuations |
| `deep` | Anthropic Opus | paid | only if you don't mind a beat of lag |

Switch anytime from the menu bar. The controller falls back across free providers before ever spending; under spend-cap lockdown every tier stays free.

## Known limits (v1)

- Some **Chrome/Electron web text fields** don't expose their contents to Accessibility. Ghostype tries to flip on web accessibility (`AXManualAccessibility`), but a few sites still hide their text — there it stays quiet. Native fields, Messages, Mail, Notes, Slack desktop, terminals, etc. all work.
- Ghost text is a floating bubble at the caret (macOS can't draw *inside* another app's text field), styled to read like inline ghost text.

## Privacy

Your **learned writing** — the n-gram phrase model and the distilled style profile — lives only in `~/.ghostype/` and is never uploaded. Predictions themselves are generated by the shared AI controller, which sends the text *before your caret* (plus your style profile) to an LLM provider to complete; on the default `fast`/`smart` tiers that's the free GitHub Models endpoint. Password/secure fields are detected and never read, logged, or sent. Delete everything any time from the menu bar → "Clear learned data". If you want fully on-device inference back, point the controller at a local model.
